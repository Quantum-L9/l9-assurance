# Legacy Source Quarry

The old repository is evidence and algorithm inventory, not a foundation.

A proposed port must record:

- source commit and file;
- invariant or algorithm being extracted;
- why assurance, rather than SDK/Core/Repair/Intelligence/LSP, owns it;
- target package and dependency impact;
- security and determinism implications;
- new conformance and adversarial tests;
- confirmation that no legacy package identity or execution authority is reintroduced.

Package identity, file count, or sunk effort is never sufficient justification.
